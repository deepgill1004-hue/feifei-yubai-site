# 文章新增固定指令

之後每一次從每日內容包生成文案或文章時，都要遵守以下規則。

## 文章挑選與改寫

1. 先判斷內容包裡哪一段最適合放到一頁式網站的文章新增功能。
2. 優先選用完整長文、方格子長文、SEO 長文，或能回答單一搜尋問題的段落。
3. 如果原文太長，可以依網站文章閱讀節奏裁切、重排段落、小標題與 CTA。
4. 如果適合電子報，就轉成電子報格式；如果不需要，不要硬轉電子報。
5. 網站文章要保留核心觀點、案例、可搜尋關鍵字與行動建議。

## 文末固定三連結

每一篇網站文章、電子報版本、方格子貼文、LINE 延伸文案，只要是完整文章或長文，都必須在文末附上這三個連結：

```
官方 LINE：
https://line.me/R/ti/p/@371arhqu

一頁式網頁：
https://deepgill1004-hue.github.io/feifei-yubai-site/

電子報：
https://sophie-yubai.beehiiv.com/
```

可以依平台語氣加一句自然承接，例如：

```
我會在 LINE 裡用更白話的方式拆，也會把完整文章與延伸整理留在網站和電子報。

官方 LINE：https://line.me/R/ti/p/@371arhqu
一頁式網頁：https://deepgill1004-hue.github.io/feifei-yubai-site/
電子報：https://sophie-yubai.beehiiv.com/
```

## 發布前檢查

- 文末是否有官方 LINE 連結
- 文末是否有一頁式網頁連結
- 文末是否有電子報連結
- 首頁或文章列表是否連到新文章
- 如果有 Beehiiv 草稿，`beehiiv/posts/*.json` 是否新增
- RSS 是否新增文章 item
